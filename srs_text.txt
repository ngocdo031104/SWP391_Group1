The flow transitions from public access (Guest) to role-specific internal dashboards upon successful authentication. Public &amp; Authentication Entrance: Guests interact with the Home Page , Search &amp; Filter Tours , and Tour Details . Any attempt to create a booking, save favorites, or access community features forces a redirect to the Login / Register Account screen. Customer Journey: Once authenticated, the Customer is routed to the Customer Dashboard . From here, they can navigate through two primary logical flows: Transaction Flow: Tour Details $\rightarrow$ Create Booking $\rightarrow$ Process Payment (incorporating Apply Coupon) $\rightarrow$ Generate Invoice. Post-Booking &amp; Engagement Flow: Booking History $\rightarrow$ Request Cancellation or Submit Tour Review; and Buddy Matching $\rightarrow$ Real-Time Chat &amp; Video Call Planning. Back-Office Dashboards (Staff, Accountant, Guide, Admin): Upon logging in, administrative and field staff bypass public flows and land directly onto their dedicated control consoles: Tour Guides utilize a mobile-responsive interface to check in participants, update live tour statuses, and report incidents. Operations Staff monitor and execute bookings, manage master guest lists, assign guides, and trigger customer alerts. Accountants supervise the financial health via transactional logs, fraud monitors, refund processing, and coupon management. Administrators possess global overrides to configure core system parameters, manage user accounts, allocate permissions, moderate content, and view predictive business analytics. 1.4.2 Screen Authorization This matrix defines the access permissions for each screen across the 6 verified system roles. Search and Filter Tours Primary Actors Guest, Customer Secondary Actors None Description As a Guest, I want to search, filter, and sort tours by destination, date, price, category, or difficulty level so that I can freely browse tour listings without making any account commitments. As a Customer, I want to search, filter, and sort tours by destination, date, price, category, or difficulty level so that I can easily find a trip that matches my specific preferences to plan a booking. Preconditions The user is on the tour listing page or interacting with the main search bar component. Tour metadata, departure schedules, activity status, and availability status are already initialized in the database. Postconditions The system displays a filtered list of tours matching the exact search and filter criteria specified by the user. Normal Sequence/Flow 1. The user enters search parameters (destination name, date, budget) in the homepage or header search bar. 2. The server sanitizes the budget parameter, queries the database using 'tourDAO.searchTours(dest, null, maxPrice, date)', and retrieves the initial matching tours list and departure schedules for remaining seats. 3. The page renders search listings at, displaying distinct destinations in the search datalist and active departure cities in the sidebar checkbox options. 4. The user refines their search using the sidebar filters (price range slider, categories, rating stars, duration, type, start city, seats availability) or selects a sorting option (price low-to-high, high-to-low, top-rated, newest). 5. The client-side script instantly filters and sorts the JSON data locally, updating the matching count, the tour grid, and the interactive map pins in real-time without reloading the page. 6. The user can toggle currency between USD and VND (rates are dynamically converted at 1 USD = 25,000 VND), click pins on the interactive map to view popup cards, or click the heart button to toggle wishlist status. Alternative Sequences/Flows A1. No Results Found: * If no tours match the user's keywords or filter combinations, the system displays a message: &quot;No tours found matching your criteria. Please try adjusting your filters.&quot; and displays a list of recommended popular tours underneath. A2. Search Input Sanitization: * If the user enters special characters or potentially malicious scripts into the search field, the system sanitizes the input, strips out invalid characters, and returns an empty results page safely without crashing.
